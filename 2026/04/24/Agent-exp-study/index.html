一个简单的Agent学习

基本组成

Agent的核心组成部分包括：

  • 模型 ：智能体的大脑，负责处理输入并生成响应；
  • 工具 ：智能体可以根据用户请求执行的预定义功能；
  • 工具箱 ：智能体可使用的工具集合；
  • 系统提示 ：指导智能体如何处理用户输入并选择正确工具的指令集。

规划

当用户提出 Query 交给 agent 之后，agent可以先进行任务规划，将总任务分化成几个子任务，增强任务处理效率

Agent会对处理结果进行反思与完善：agent对历史动作可以自我批评和自我反思，从错误中学习并在后续步骤里完善，从而改善最终结果的质量。

工具

Agent会根据不同的任务来选择不同的工具进行使用。普通的 LLM 大模型是无法调用外部工具链的，Agent的加入可以使得LLM学习调用外部API获取额外信息，包括当前信息、代码执行能力、专有信息源的访问等等

行动

行动模块是智能体实际执行决定或响应的部分。面对不同的任务，智能体系统有一个完整的行动策略集，在决策时可以选择需要执行的行动，比如广为熟知的记忆检索、推理、学习、编程等。

记忆

  • 短期记忆：上下文学习即是利用模型的短期记忆学习
  • 长期记忆：为agent提供保留和召回长期信息的能力，通常利用外部向量存储和检索实现

Agent 执行工作的流程可以简单总结为：

  1. 接受用户输入后，询问 AI Agent，然后Agent到LLM那里处理，生成相应的回复
  2. 得到回复后，Agent开始做决策/规划，然后决定是否选择工具，以及选择使用什么工具
  3. 使用工具后，将得到的结果返回Agent，然后Agent将最终的请求发回给用户。

事实上，了解了Agent的定义，仍然不是很清楚，Agent是如何和LLM在一起使用的，所以在网上找了一个简单的Agent的例子，用来学习理解

Agent 实例

整个实例代码的结构包含以下几个部分：

  • 类 OllamaModel ，相当于将和大模型交互的部分给包装起来了。
  • 工具函数： basic_calculator 和 reverse_string
  • 类 ToolBox ，我理解的就是将工具整合打包起来，方便Agent进行调用，即把工具描述整理为 prompt 文本。
  • agent_system_prompt_template ：这是整个Agent最核心的部分，作为一个小协议/规则的形式存在，告诉大模型不是直接输出结果给用户，而是整合成规定的形式交给Agent，由Agent来判断是否使用工具，以及之后如何处理。
  • 类 Agent ：是实现整个Agent最核心的类，定义了如何与大模型，工具集之间交互，并且规定了如何输出。
  • main ：整个程序的主要逻辑，把上面的函数/类做成一个可以和用户进行交互的程序。

类 OllamaModel

初始化

初始化定义了大模型的API接口，模型名称，以及使用的一些模型相关的参数，如： system prompt 以及消息头等。

函数 generate_text

根据提示词，以及一些请求中信息的定义组成一个POST请求，向大模型发起对话，并拿到返回内容。

工具函数

basic_calculator 工具

首先对输入的格式进行判断和清洗。这里说明一个很重要的工程思想：工具不能假设模型永远输出正确格式。然后如果满足规定的格式，就会用对应的函数得到结果，并且返回。

reverse_string 工具

输入必须是一个字符串，然后最后返回字符串的逆序输出结果

类 ToolBox

这个类，我把它简单地理解为 Agent 和 LLM 之间进行交互的一个媒介。它将 Agent 可以使用的工具全部打包起来，然后将其整理为 prompt 文本，作为 system_prompt 交给大模型。

agent_system_prompt_template

这一系统提示词告诉模型不是直接回答用户，而是先输出“我要调用哪个工具，用什么参数”。这和普通聊天最大区别在于：

  • 聊天模型输出的是自然语言
  • Agent 模型输出的是动作指令

类 Agent

这个类初始化规定了要使用的 model ，模型名字以及停止输出的标志。

prepare_tools 初始化了要使用的工具以及对应描述，用于嵌入 system_prompt 。

think 函数将用户输入的prompt提供给LLM，然后得到LLm的输入

work 函数将 think 函数得到的返回值进行处理，如果有使用工具的话，就调用相应的工具函数并且得到返回值。

main 函数

初始化模型，打印Agent相应使用工具信息，然后获得输入，并且进行处理得到回应值。工作流：

agent -> work -> think -> LLM -> response

response 返回给 work，然后调用工具集对返回值进行处理。

以上内容是一个最基本的 agent 调用工具，与LLM结合的例子。
