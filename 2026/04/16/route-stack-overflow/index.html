route_stack_overflow

复现了某路由器上一个栈溢出vuln的POC，本身没什么难度，但是环境搭建的时候学习到了一点东西。

环境

MacOS arm + UTM （Linux arm架构虚拟机）

文件处理

  • Binwalk是一款功能强大的固件分析工具，能够快速识别和提取嵌入在文件中的数据，广泛应用于固件分析、嵌入式系统研究和安全审计等领域。

下载固件之后，用 binwalk 进行分离（需要多次分离），得到两个文件：

SquashFS is a highly compressed, read-only filesystem for Linux, designed for archival, embedded systems, and live CD/USB storage

在虚拟机中对 partition_1.bin 用 squashfs 进行处理，然后ida逆向分析 /bin/httpd 文件

查看函数formSafeEmailFilter ，找到对应的漏洞点，现在在IDA中追寻一下调用链条，方便写POC / EXP。

sub_6B948 at 0x6B948，这里初始化 Web 路由，0x6BBCC 注册 “/goform” 对应处理函数 websFormHandler0x6BC20 调用 sub_2DB50()，开始批量注册各个 form handler

sub_2DB50 at 0x2DB50，这是 form 注册函数集合。0x2E764 这一行注册”SafeEmailFilter”，函数：formSafeEmailFilter (0x55480)，sub_14914(“SafeEmailFilter”, formSafeEmailFilter);

  1. sub_14914 at 0x14914，它把 “SafeEmailFilter” 和函数指针 formSafeEmailFilter 插入到全局 form 表 dword_925A0 中

websFormHandler at 0x147A0，处理 “/goform/…” 请求。它从 URL 中取出 form 名，调用 sub_1D28C(dword_925A0, s) 查表。查到后在 0x148EC 通过函数指针 v14(…) 间接调用目标 handler。

所以这个函数的实际触发入口是：

/goform/SafeEmailFilter

如果请求里再带上参数 page=，来造成缓冲区溢出。

还使用本地虚拟机吧😭

在运行 /bin/httpd 服务之前，需要先将虚拟机内的网络环境搭建好，使得路由器能够正常工作。所以需要：

  • 一张给固件使用的虚拟网卡
  • 一个二层交换环境
  • 一个能拿到 IP 的网络
  • 让固件里的 httpd 觉得“LAN 已经起来了”

桥接网络

桥接网络是指本地物理网卡和虚拟网卡通过VMnet0虚拟交换机进行桥接，物理网卡和虚拟网卡在拓扑图上处于同等地位，那么物理网卡和虚拟网卡就相当于处于同一个网段，虚拟交换机就相当于一台现实网络中的交换机,所以两个网卡的IP地址也要设置为同一网段。

这里可能需要对Linux的桥接网络，虚拟网卡，物理网卡，桥，网卡接口，IP等概念以及之间的关系有比较深刻的理解。

其实，从Linux内核3.x版本开始，物理网卡和虚拟网卡是平等的设备，它们都会在注册时创建net_device数据结构来保存(物理或虚拟)设备信息。

相比于物理网卡负责内核网络协议栈和外界网络之间的数据传输，虚拟网卡的两端则是内核网络协议栈和用户空间，它负责在内核网络协议栈和用户空间的程序之间传递数据：

为了使得路由器能够正常工作，首先得把它当作一个真实物理设备，需要一根网线连接到外部网络，所以先新建一个虚拟网卡 tap0 ，然后与该固件设备里的路由连接，虚拟网卡需要进一步连接外部网络，所以就需要先连接一个桥，然后该桥与以太网接口连接，从而连接到所谓的外部网络。在本实验环境中，该桥与以太网连接后，UTM虚拟机的以太网接口接入了宿主机的网络。

一些命令相关的知识：

brctl is used to set up, maintain, and inspect the ethernet bridge configuration in the Linux kernel.

因为 tap0 和 enp0s1 都只是“网卡接口”，它们本身不会充当交换设备。要让两个二层接口真正互通，需要一个负责转发以太网帧的中间体，br0 就是这个角色。

tap0 像一个网口，enp0s1 也像一个网口，两个网口不能直接“插”在一起让 Linux 自动帮你转发所有二层流量，需要一个“交换机”来学习 MAC、转发广播包、ARP、DHCP 等二层数据。br0 就是 Linux 里的这个虚拟交换机

为什么必须是二层交换，而不是别的方式：

  • 路由器固件启动依赖的通常不是单纯 IP 连通，而是完整二层环境
  • DHCP 发现、ARP 广播、接口初始化这些都依赖二层帧转发
  • br0 能把 tap0 和 enp0s1 放进同一个广播域
  • 这样固件才会觉得“自己真的插在一个网里”
